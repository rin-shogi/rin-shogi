plan の Phase G(M7、新設)。本端末で R3300 帯到達後、R3500+ を本気で狙うための
計算資源戦略。**「クラウドで効果検証 → ハード購入判断」** の順で投資リスクを最小化する。

## 戦略の3段階

```
段階 G1: クラウド CPU 試行(月 3-10 万円 × 1-3 ヶ月)
   ↓ 効果が見えた?
段階 G2: 投資判断
   ├ a. クラウド継続(月予算継続)
   ├ b. 自作機購入(ミドル ~50万円, ハイ ~100万円)
   └ c. DL系へ転換(別チケット化候補)
   ↓
段階 G3: R3500+ 着手
```

## G1: クラウド CPU 試行

### 候補プロバイダ

| プロバイダ | 想定インスタンス | スポット時間単価 | 用途 |
|---|---|---|---|
| AWS | c7i.16xlarge (32 vCPU) | 約 $1.0/hr | 教師生成・学習・並列自己対戦 |
| AWS | c7a.16xlarge (32 vCPU AMD) | 約 $0.9/hr | 同上(AMD ベース) |
| GCP | c3-highcpu-44 | 約 $1.1/hr | 同上 |
| Vast.ai (CPU) | 32-64 vCPU | $0.30-0.60/hr | 個人向け最安 |
| Lambda Labs | CPU インスタンス | 要見積 | NNUE には実質不要 |

> 月予算の目安: AWS spot c7i.16xlarge を **8時間/日 × 30日 ≒ $240/月 ≒ 4万円**。
> 本気で回すなら 24時間 × 30日 ≒ $720 ≒ 11万円。

(対応するプロビジョニングスクリプトは G1 着手時に追加)

### G1 で実測する項目

- 教師生成スループット(局面/時間)を本端末比で計測 → スケール係数を把握
- 1 サイクル(100万→1000万→1億局面)の所要時間
- 自己対戦並列数を本端末より上げた効果(N=1000局を 1日で回せるか 等)
- 月予算 vs floodgate レーティング向上の感度(費用対効果)

## G2: 投資判断

### 判定基準

| 観測 | 判定 |
|---|---|
| クラウド試行で floodgate +100 Elo 以上の上昇 + 上昇余地が見える | **b. 自作機購入** または **a. クラウド継続**(4ヶ月以上の継続見込みなら b) |
| クラウド試行で +50 Elo 程度、スケールでさらに伸びる感触 | **a. クラウド継続** で更にデータを取る |
| クラウド試行でも明確な上昇が見えない | **c. DL系転換**(別チケット化)を検討 |
| 予算が逼迫 | 一旦撤退、本端末完結で運用継続(R3300 で安定化) |

### 自作機購入の候補(2026年5月時点)

| ティア | GPU | 構成 | 総額 |
|---|---|---|---|
| エントリー | RTX 4060 Ti 16GB | Ryzen 7700X / 64GB / 1TB / 750W | 約 25-30 万円 |
| ミドル(推奨) | RTX 5070 Ti 16GB | Ryzen 9 9950X / 96GB / 2TB / 850W | 約 45-55 万円 |
| アッパーミドル | RTX 5080 16GB | Ryzen 9 9950X / 128GB / 2TB / 1000W | 約 60-75 万円 |
| ハイエンド | RTX 5090 32GB | Threadripper 7960X / 128GB / 4TB / 1200W | 約 100-130 万円 |

> 中古 RTX 4090 24GB(~25-30万円)+ 自作 PC(~25万円)で **総額 ~50万円** も有力候補。
> NNUE 系の文脈では VRAM より CPU コア数の方が効くため、
> **エントリー〜ミドル GPU + 強力 CPU(16-32コア)** がコスパ良い。

### クラウド vs 購入の損益分岐(参考)

- ミドル機 ~50万円 ≒ AWS c7i.16xlarge spot で **22-25 ヶ月** の常時稼働相当
- 8時間/日 × 30日運用なら **約 65 ヶ月**、24時間運用なら **約 22 ヶ月**

つまり「2 年以上回し続ける確信があるなら買い、それ未満ならクラウド」が大雑把な目安。

## G3: R3500+ 着手

拡張リソースを得たら、Phase D/F のレシピを拡張規模で再実行:

1. 教師生成: 1億 → 10億〜100億 局面
2. 学習: 1サイクル → 数サイクル、レシピを段階的に
3. 探索チューニング: パラメータ数を増やす、SPSA 試行数を増やす
4. 定跡: 大規模 DB 構築(局面数 数十万〜)
5. floodgate 連続運用、レーティング推移評価

R3500 を超えてさらに上を狙う場合は、本チケット範囲外(別チケット化、DL系転換も含めて再設計)。
